# Home/Dashboard Page Test Cases (Client Interactable Features)

## 1. UI Elements

- [ ] Home/Dashboard loads successfully after login
- [ ] Welcome message or user info is displayed
- [ ] Recent jobs section is visible
- [ ] Recent posts/feed section is visible
- [ ] Matched jobs section is visible (for taskers)
- [ ] Navigation bar is present
- [ ] Notifications icon is present
- [ ] Profile/avatar is visible in navbar
- [ ] Search bar is present (if applicable)
- [ ] Quick links/buttons to main modules (Profile, Feed, Network, Jobs, Messages)

## 2. Data Loading

- [ ] Recent jobs are fetched and displayed
- [ ] Recent posts are fetched and displayed
- [ ] Matched jobs are fetched and displayed (for taskers)
- [ ] Loading indicator appears while data is loading
- [ ] Error message shown if data fails to load

## 3. Navigation

- [ ] Clicking Profile navigates to Profile page
- [ ] Clicking Feed navigates to Feed page
- [ ] Clicking Network navigates to Connections page
- [ ] Clicking Jobs navigates to Jobs page
- [ ] Clicking Messages navigates to Messages page
- [ ] Clicking Notifications icon opens dropdown or navigates to Notifications page

## 4. Interactions

- [ ] Can click on a job to view job details
- [ ] Can click on a post to view post details
- [ ] Can click on matched job to view details (for taskers)
- [ ] Can use search bar to search jobs/posts/users (if applicable)

## 5. Auth State

- [ ] If not authenticated, user is redirected to login page

## 6. Accessibility

- [ ] All main elements have proper labels
- [ ] All interactive elements are keyboard accessible

## 7. Edge Cases

- [ ] Dashboard loads with no jobs/posts and displays empty state message
- [ ] Dashboard loads with large number of jobs/posts and handles gracefully
- [ ] Multiple rapid navigation actions are handled correctly
